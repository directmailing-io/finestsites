DevOps-Experte

Server, Docker, Deployment, Betrieb

Coolify auf Hetzner ist eine solide Wahl für diese Größe — einfach zu bedienen, Git-Integration, automatisches SSL. Das passt.

Das Problem ist GoTrue selbst zu hosten. GoTrue ist das Login-System von Supabase — Open Source, gut, aber komplex im Betrieb. Wenn Supabase eine Sicherheitslücke schließt, muss ich das manuell auf deinem Server einspielen. Passiert das nicht rechtzeitig, ist das ein ernstes Sicherheitsproblem. Supabase hat ein Team dafür. Wir sind zwei.

Mein Vorschlag: BetterAuth statt GoTrue. Modernes Open-Source-Projekt, läuft auf deiner eigenen Datenbank, hervorragende Next.js-Integration, aktiv gepflegt. Erfordert ca. 2–3 Tage Code-Anpassungen — aber dann haben wir ein wartbares System.

Urteil GoTrue raus, BetterAuth rein. Coolify + Hetzner bleiben.
🔐

Security-Experte

Datensicherheit, Angriffsflächen, DSGVO

Ich stimme beim DevOps-Kollegen zu — GoTrue selbst hosten bedeutet, dass wir selbst für Sicherheits-Patches verantwortlich sind. Das ist kein theoretisches Risiko.

Zusätzlich: PostgREST ist gefährlich in falschen Händen. PostgREST stellt die gesamte Datenbank als Web-API bereit. Die Sicherheit hängt an "Row Level Security"-Regeln — eine einzige falsch konfigurierte Regel bedeutet, dass Kundendaten für alle sichtbar sind. Supabase hat Experten, die das prüfen. Wir müssen das selbst richtig machen.

Wenn wir sowieso Code anfassen: PostgREST raus, direkter Datenbankzugriff aus Next.js via Drizzle ORM. Typsicher, keine versteckten Sicherheitslücken durch falsche Regeln, jeder Next.js-Entwickler versteht es sofort.

Außerdem zwingend: Cloudflare Pro (20 $/Monat) mit WAF einschalten. Schützt vor SQL-Injection, Brute-Force auf Login, DDoS. Für eine SaaS mit Zahlungen kein Luxus.

Urteil PostgREST raus, Drizzle ORM rein. Cloudflare WAF ist Pflicht.

Datenbank-Experte

PostgreSQL, Performance, Backups, Skalierung

PostgreSQL ist die richtige Wahl — das ist klar. Die Frage ist: selbst hosten oder managed?

Selbst hosten bedeutet: Ich bin verantwortlich für Autovacuum-Tuning, langsame Abfragen überwachen, Major-Versions-Updates (PostgreSQL 17 → 18), Backup-Tests. Das ist Fachwissen, das regelmäßig gebraucht wird.

Hetzner Managed Databases löst das: automatische Backups, automatische Sicherheitsupdates, optionales High-Availability-Failover (zweiter DB-Server übernimmt automatisch wenn der erste ausfällt). Kostet ~20–35 € statt ~8 €/Monat. Dafür: kein Datenbankadministrations-Aufwand.

Für Skalierung wichtig: Managed Databases unterstützt Read Replicas — wenn die App wächst, können Leseoperationen auf einen zweiten Datenbankserver ausgelagert werden. Bei self-hosted müsste ich das manuell einrichten.

Urteil Hetzner Managed Databases statt self-hosted PostgreSQL. ~25 € mehr pro Monat, dafür professionell gewartet.
📈

Skalierungs-Experte

Architektur, Load Balancing, Wachstum

Kundenseiten laufen über Cloudflare Workers — die skalieren automatisch auf Millionen von Anfragen. Das ist perfekt und braucht nichts geändert zu werden.

Das Dashboard (Next.js) läuft auf einem einzelnen App-Server. Wenn wir skalieren wollen, brauchen wir irgendwann mehrere App-Server hinter einem Load Balancer. Einen Load Balancer im Nachhinein einzubauen ist Arbeit und Downtime.

Hetzner Load Balancer von Anfang an: kostet 5 €/Monat. Dafür kann jederzeit ein zweiter App-Server hinzugefügt werden ohne die Architektur umzubauen. Das ist die günstigste Versicherung gegen Skalierungsprobleme.

Next.js ist von Natur aus horizontal skalierbar — jede Instanz ist unabhängig. Passt perfekt zu Load Balancer + mehrere Server.

Urteil Hetzner Load Balancer (5 €/Monat) von Anfang an. Billigste Entscheidung mit dem größten Nutzen.

SaaS-Experte

Kosten, Betrieb, Business-Realität

Ich will ehrlich sein: BetterAuth + Drizzle ORM einzuführen ist kein "Umzug" mehr — das ist eine Teilumschreibung des Backends. Rund 3 Tage Entwicklungsarbeit, in denen keine neuen Features entstehen. Das muss bewusst entschieden werden.

Macht es sich bezahlt? Ja — weil GoTrue + PostgREST langfristig mehr Kopfschmerzen verursachen als die 3 Tage jetzt kosten. Technische Schulden wachsen. Lieber jetzt den sauberen Schnitt.

Die Kostenbilanz stimmt: Vercel (~20 $/Monat) und Supabase (free oder 25 $/Monat) fallen weg. Hetzner kostet ~25–35 €/Monat. Je nach aktuellem Plan sparst du sofort oder in Kürze. Bei 100+ zahlenden Kunden rechnet sich das deutlich.

Urteil Den Schnitt jetzt machen. Technische Schulden heute sind morgen doppelt teuer.
