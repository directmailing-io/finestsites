Was wir selbst hosten (die drei Teile)

💡
Wichtig zu verstehen: Supabase macht eigentlich drei Dinge für dich: Datenbank, Login und eine Schnittstelle für die App. Wir ersetzen alle drei auf Hetzner — der App-Code ändert sich dabei kein bisschen.
DB

PostgreSQL — die Datenbank

Alle deine Daten (Nutzer, Sites, Templates, Formulare) liegen in einer PostgreSQL-Datenbank. Die exportieren wir bei Supabase und importieren sie auf Hetzner. Das ist der einfachste Teil.

Wie ein USB-Stick: Daten raus, Daten rein.

Auth

GoTrue — das Login-System

Supabase nutzt intern ein Programm namens "GoTrue" für Anmeldungen, Passwörter und Sessions. Das ist Open Source und läuft als kleines Docker-Programm auf Hetzner. Verhält sich exakt gleich wie bei Supabase.

Eine Subdomain wird benötigt, z.B. auth.finestsites.de

API

PostgREST — die Datenschnittstelle

Die App spricht nicht direkt mit der Datenbank, sondern über eine Schnittstelle (API). PostgREST stellt diese Schnittstelle bereit — auch das läuft als Docker-Programm, auch das ist Open Source.

Eine Subdomain wird benötigt, z.B. api.finestsites.de

Was du beachten musst

⚠️
Backups: Bei Supabase passieren Backups automatisch. Auf Hetzner richten wir das selbst ein — täglicher automatischer Export in den Hetzner Object Storage (~5 €/Monat). Das machen wir von Anfang an, bevor wir den DNS umstellen.
⚠️
Updates: Supabase hält sich selbst aktuell. Auf Hetzner bist du selbst dafür verantwortlich, dass Sicherheitsupdates eingespielt werden. Coolify macht das für die App automatisch. Für den DB-Server empfehle ich unattended-upgrades (automatische Sicherheitsupdates) — richte ich beim Setup ein.
⚠️
Subdomains: Wir brauchen zwei Subdomains — eine für das Login-System, eine für die Datenschnittstelle. Diese müssen bei Cloudflare eingerichtet werden (ich mache das, du musst nur OK geben). Z.B. auth.finestsites.de und api.finestsites.de.
🛡️
Kundenseiten bleiben immer online: Sie laufen über Cloudflare Workers und sind vollständig unabhängig vom Hetzner-Server. Selbst wenn der Server ausfällt, sind alle Kundenseiten erreichbar.

Was du tun musst (7 Schritte)

1

App-Server anlegen Pflicht

Hetzner Cloud → Neues Projekt → Server erstellen:
System: Ubuntu 24.04 · Typ: CX32 · Region: Nürnberg · SSH-Key hinzufügen

2

Datenbank-Server anlegen Pflicht

Zweiter Server: System: Ubuntu 24.04 · Typ: CX22 · Region: Nürnberg · gleicher SSH-Key
Dann: Private Network zwischen beiden Servern anlegen (damit sie sicher miteinander reden können, ohne über das Internet zu gehen)

Private Network: Hetzner Cloud → Networks → Netzwerk erstellen → beide Server hinzufügen.

3

Hetzner Object Storage anlegen Pflicht

Für die automatischen Datenbank-Backups: Hetzner Console → Object Storage → Bucket finestsites-backups erstellen → Access Key + Secret Key notieren und mir schicken.

Kostet ca. 5 €/Monat und enthält 30 Tage Backups.

4

Coolify installieren Pflicht

Per SSH auf den App-Server einloggen und diesen Befehl ausführen:

curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash

Danach: Browser öffnen → http://SERVER-IP:8000 → Admin-Account anlegen.
Dann: Coolify URL + API Token (unter Einstellungen → API) an mich schicken.

5

SSH-Zugang schicken Pflicht

Ich brauche SSH-Zugang zu beiden Servern — entweder den privaten SSH-Schlüssel oder ein temporäres Root-Passwort.

6

Datenbank-Export aus Supabase Pflicht

Supabase Dashboard → Settings → Database → Backups → "Download latest backup"
Die Datei (endet auf .sql.gz oder .dump) an mich schicken.

Das sind alle deine Daten — Nutzer, Sites, Formulare, Templates. Vertraulich behandeln!

7

Umzugs-Zeitfenster + Subdomains bestätigen Pflicht

Bitte beantworte folgende Fragen:
— Wann darf das Dashboard für ca. 1 Stunde offline sein? (z.B. Sonntag 3:00 Uhr)
— Soll ich auth.finestsites.de und api.finestsites.de als Subdomains nutzen — oder andere Namen?
— Supabase danach direkt kündigen, oder erst 2 Wochen parallel laufen lassen?

Was ich dann erledige

Phase 1 — Server einrichten

PostgreSQL auf dem DB-Server installieren, absichern, Datenbank-Dump importieren. PgBouncer für Verbindungspool. Automatische tägliche Backups zu Hetzner Object Storage. Firewall: DB nur intern erreichbar.

ca. 1–2 Stunden · kein Downtime

Phase 2 — Login-System und API aufsetzen

GoTrue (Login) und PostgREST (Datenschnittstelle) als Docker-Container auf dem App-Server. Subdomains auth.finestsites.de und api.finestsites.de einrichten mit SSL. Alles testen.

ca. 1 Stunde · kein Downtime

Phase 3 — App deployen und testen

Next.js über Coolify deployen mit allen neuen Einstellungen (neue DB, neues Login-System). Vollständiger Test unter temporärer URL: Login, Stripe, Editor, Publizieren, Custom Domains.

ca. 1 Stunde · kein Downtime

Phase 4 — DNS umstellen (Downtime-Fenster)

Letzter Daten-Sync. Dashboard kurz offline setzen. DNS auf Hetzner umbiegen. SSL-Zertifikat. Stripe Webhook aktualisieren. Smoke-Test aller wichtigen Funktionen.

ca. 30–60 Minuten Downtime für das Dashboard · Kundenseiten bleiben online

Phase 5 — Stabilisierung

Uptime-Monitoring einrichten. Backup-Wiederherstellung testen. Nach 1–2 Wochen stabilem Betrieb: Vercel kündigen, Supabase kündigen.
