Team-Anmerkungen: Was muss umgesetzt werden?

Conversion-Strategie

  • 3 CTA-Punkte: Sticky Nav, Hero rechts, Bottom — immer erreichbar
  • Pins (1, 2, 3): Highlight-Klick = zugehoeriger Pin pulsiert → Aufmerksamkeit gelenkt
  • Editor zuerst erleben, dann kaufen — Aktivierung vor Commitment
  • Einwaende aktiv eliminieren: "5 Min.", "kein Code", "kostenlos testen"
  • Trust: Sterne, "127 aktive Websites", Testimonials mit konkreten Namen

Technische Anforderungen

  • Interactive Preview: postMessage → iframe (bereits implementiert!)
  • Neues DB-Feld: templates.preview_config (JSONB)
  • Preview-URL: /api/templates/[id]/public-preview?data=base64
  • Admin-Panel: preview_config Editor pro Template
  • Sections: data-section Attribute + Toggle-API
Next.js postMessage base64 JSON Cloudflare R2

Content pro Template (Admin pflegt)

  • Highlights (1-5): 4-6 Bullet Points, was dieses Template besonders macht
  • Demo-Daten: Beispiel-Name, Texte, vorgefuellte Felder
  • Editierbare Felder: Welche zeigt der Editor? (Farbe, Bild, ...)
  • Editierbare Sektionen: Welche sind toggle-bar?
  • Testimonials: 3 Stimmen pro Template, zielgruppenspezifisch

Umsetzungs-Reihenfolge

  • Phase 1 (1 Woche): Neues Split-Hero-Layout + Highlights + "Wie es funktioniert" + Social Proof
  • Phase 2 (1 Woche): Farbtheme-Picker live (postMessage)
  • Phase 3 (1 Woche): Section-Toggles + Header-Bild-Auswahl
  • Phase 4: Social Proof dynamisch aus DB + A/B-Test CTAs
