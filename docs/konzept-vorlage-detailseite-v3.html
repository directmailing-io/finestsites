3 — NEU: Interaktiver Editor-Preview (Herzstuck)

Apple-Idee: Sticky CTA-Banner erscheint, sobald man den Hero-Bereich scrollt ("Bereit? → Template starten") — wie Apples Sticky Checkout Bar.

Neue Features — Eigenstaendige Ideen
A) Sticky Purchase Bar (wie Apple): Sobald der Hero aus dem Viewport scrollt, erscheint unten eine schmale Bar: Template-Name links, "Template starten →" rechts. Verschwindet wieder bei Bottom-CTA. → Immer sichtbarer CTA.

B) Editor: Vorschau-Tabs auf Mobile: Oben zwei Tabs "Vorschau" | "Anpassen". Nutzer switcht zwischen scrollbarer Template-Ansicht und Editor-Controls. Beide Panes sind full-height. → Kein Scrollen auf Mobile um zwischen Preview und Controls zu wechseln.

C) "Weitere Templates" nach Netzwerk gefiltert: Statt random 3 Templates: immer nach Tags gefiltert (gleiche Firma). Query: WHERE tags && ARRAY['{tag}'] AND id != current_id LIMIT 3. Falls zu wenig: fallback auf allgemeine NM-Templates. → Relevanz statt Zufall.

D) Editor CTA unten in der Sidebar: Nach den Controls direkt ein "Jetzt starten" Button innerhalb des Editors. Fuhlt sich natuerlich an nach dem Ausprobieren. → Conversion im richtigen Moment.

E) Hero: Cover-Bild entfernt — Editor IST das Cover: Kein statisches Screenshot mehr nach dem Hero. Der Editor-Bereich (scrollbar, interaktiv) ist selbst der erste Eindruck vom Template. Das ist viel ueberzeugender als ein flaches Bild. → Sehen = Kaufen.
